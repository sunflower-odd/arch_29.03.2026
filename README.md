## Сделанные оптимизации

### По памяти
- убрала загрузку в память текста целиком, вместо этого используем потоковую обработку и генератор;
- удалила лишние промежуточные структуры, так как теперь читаем данные с помощью генератора.


### По скорости
- убрала лишнюю функцию с множественными запросами к URL, достаточно одного запроса;
- используем set вместо dict - дубликаты не нужны, а также проверка w in words_to_count O(1) вместо O(n);
- оптимизировала циклы прохода по данным: проходим 1 раз по словам с сайта, а не сначала по словам из файла, а потом по словам с сайта, это снижает сложность с O(n*m) до O(n);
- добавила таймайут в запрос к сайту, в случае проблем с ответом, не будет зависать, а просто вернет ошибку.

### Прочее
- добавила сплит данных с сайта по другим символам, так как тут возвращается HTML код, который содержит различные разделители
- в целом, тут парсится HTML код, не исключено, что это ошибка и требуется только текст, видимый пользователям страницы.
Однако, в задании сказано, что нужно оптимизировать код, но не ставить под сомнение требуемый функционал.
- исправила мелкие недочеты: пусть функция возвращает значение, также пусть ошибки будут видны через raise_for_status()
